Estudo de Viabilidade: Workflow em Blocos Composiveis

Analise completa — catalogo de blocos, visao do tecnico, visao do gestor, arquitetura

1. Catalogo Completo de Blocos 38 BLOCOS
Organizados em 6 categorias por cor. Cada bloco tem um papel especifico e se encaixa em slots compativeis.
⚙
FLUXO (Controle)
7 blocos — definem a logica do caminho
INICIO Ponto de partida (fixo, 1 por fluxo)
FIM Encerra o fluxo (pode ter varios: Sucesso, Falha, Reagendar)
SE Avalia condicao → abre 2 caminhos (SIM / NAO)
SENAO SE Encadeia apos SE para condicoes multiplas
ENQUANTO Repete blocos internos enquanto condicao for verdadeira
REPETIR Repete N vezes (encaixa bloco NUMERO)
IR PARA Pula para outro bloco (loops, voltar etapa)
✍
ACOES (Tecnico executa)
9 blocos — o que aparece na tela do tecnico
ETAPA Passo generico que o tecnico confirma (nome + descricao)
FOTO Tirar/enviar foto (antes, depois, evidencia)
ASSINATURA Capturar assinatura digital do cliente no celular
FORMULARIO Campos customizaveis que o gestor define
CHECKLIST Lista de itens para marcar (check, check, check)
NOTA Campo de texto livre para observacao
GPS Registra localizacao GPS atual
SCANNER Ler codigo de barras / QR Code
PERGUNTA Exibe pergunta com opcoes (SIM/NAO, lista, texto)
🔢
DADOS (Valores)
7 blocos — encaixam dentro de outros blocos como parametros
NUMERO Valor numerico editavel (ex: 100 metros, 30 minutos)
TEXTO Texto editavel (mensagem, template, nome)
SIM/NAO Verdadeiro ou Falso (toggle)
LISTA Opcoes pra escolher (dropdown customizavel)
DATA/HORA Data, hora ou periodo (ex: "amanha 14:00")
VARIAVEL Referencia a campo anterior (resposta de pergunta, etc)
CAMPO OS Dados da OS (valor, endereco, cliente, tecnico)
❓
CONDICOES (Encaixam no SE)
8 blocos — hexagonais, encaixam apenas nos slots de condicao
IGUAL A Campo = Valor
DIFERENTE Campo ≠ Valor
MAIOR QUE Numero > Numero
MENOR QUE Numero < Numero
CONTEM Texto contem substring
ESTA PERTO GPS dentro de X metros do endereco da OS
HORA ENTRE Hora atual entre X e Y (horario comercial, etc)
TEM Verifica se existe (foto tirada? assinatura? resposta?)
💬
COMUNICACAO
4 blocos — mensagens automaticas e aprovacoes
NOTIFICAR Envia WhatsApp/SMS automatico (nao trava o fluxo)
APROVACAO TRAVA o fluxo ate o gestor aprovar (gate)
ALERTA Dispara alerta no dashboard do gestor
EMAIL Envia email formatado (relatorio, resumo)
⚙
SISTEMA
6 blocos — temporizadores, status, calculos
DELAY Aguarda X minutos/horas (encaixa bloco NUMERO)
SLA Define tempo limite — dispara ALERTA se estourar
STATUS Muda status da OS automaticamente
REAGENDAR Reagenda a OS para outra data (encaixa DATA/HORA)
CALCULAR Operacao matematica (soma, multiplica, etc)
FINANCEIRO Gera lancamento financeiro / simula comissao

5. Analise de Viabilidade

✔ VIAVEL — Sim, da pra fazer

  • Retrocompativel: Fluxos atuais sao apenas blocos ETAPA em sequencia — zero quebra
  • JSON flexivel: Blocos como grafo (id + next) em vez de array linear
  • Motor simples: Interpretador que caminha pelo grafo, bloco por bloco
  • Tecnico nao ve complexidade: So ve uma tela por vez
  • Sem dependencias externas: Tudo React + SVG, sem library de diagramas
  • Templates pre-prontos: Gestor escolhe template e customiza

⚠ Cuidados na implementacao

  • Loop infinito: Validar que ENQUANTO tem condicao de saida
  • Drag & drop: Implementar com cuidado no mobile (touch events)
  • Persistencia: Cada OS salva o estado do fluxo (qual bloco esta)
  • Versioning: Se gestor edita fluxo, OS em andamento mantém versao antiga
  • Rollback: Gestor pode resetar passo e mandar tecnico refazer
  • Fases: Implementar em etapas (primeiro blocos basicos, depois condicoes)

📅 Sugestao de faseamento

FASE A — Basico
  • Blocos: ETAPA, FOTO, NOTA, GPS
  • Fluxo linear (sem condicao)
  • Builder visual novo
  • Motor com estado por OS
FASE B — Logica
  • Blocos: SE, PERGUNTA, DELAY, SLA
  • Ramificacao SIM/NAO
  • APROVACAO (gate)
  • NOTIFICAR (WhatsApp)
FASE C — Avancado
  • Blocos: FORMULARIO, CHECKLIST, ASSINATURA
  • ENQUANTO, REPETIR (loops)
  • CALCULAR, VARIAVEL
  • Templates pre-prontos
